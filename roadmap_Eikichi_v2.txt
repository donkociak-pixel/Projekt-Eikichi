4. KLUCZOWE FUNKCJE

brain.py:
  - klasyfikacja(user_prompt) → jedno słowo: greeting/question/exercise/answer/other
    * think=False, num_predict=5-20, prompt po angielsku
    * Bezstanowy — bez historii, bez persony
  - zapytanie(user_prompt) → wykonanie zadania od Gemini/app.py
    * think=False, num_predict=8192, prompt po angielsku
    * System prompt: "You are a task executor. Complete the given task precisely."
    * Bezstanowy — dostaje polecenie, wykonuje, zwraca wynik

communicator.py:
  - onizuka(user_prompt) → odpowiedź Gemini jako nauczyciel
    * Gemini z personą przez chats.create(system_instruction=persona)
    * Automatyczna historia rozmowy (Gemini sam trzyma)
    * send_message() — wysyła wiadomość, zwraca response.text

app.py (dyrygent):
  - Pętla while True: input → klasyfikacja → routing
  - greeting/question/other → onizuka() sam odpowiada
  - exercise → [ChromaDB] → zapytanie() generuje wariant → onizuka() formatuje
  - answer → zapytanie() sprawdza vs wzorzec → onizuka() komentuje

5. PRZEPŁYW DANYCH (ZAKTUALIZOWANY)

--- Proste pytanie ---
1. Uczeń pisze: "wytłumacz mi logarytmy"
2. Qwen klasyfikuje → "question" (1 sek)
3. app.py → onizuka("wytłumacz mi logarytmy")
4. Gemini odpowiada jako nauczyciel (2-3 sek)
5. Uczeń widzi wyjaśnienie

--- Prośba o zadanie ---
1. Uczeń pisze: "daj mi zadanie z logarytmów"
2. Qwen klasyfikuje → "exercise" (1 sek)
3. app.py → memory.py szuka w ChromaDB "logarytmy" → 4 chunki
4. app.py → zapytanie("Generate variant of this problem: [chunk]") (2 sek)
5. Qwen generuje wariant ze zmienionymi wartościami
6. app.py → onizuka("Oto zadanie dla ucznia: [wariant]")
7. Gemini formatuje ładnie, dodaje zachętę (2 sek)
8. Uczeń widzi nowe zadanie oparte na prawdziwym arkuszu

--- Sprawdzenie odpowiedzi ---
1. Uczeń pisze: "moja odpowiedź to x=5"
2. Qwen klasyfikuje → "answer" (1 sek)
3. app.py → zapytanie("Check: student=x=5, correct=x=5. Same?") (1 sek)
4. Qwen porównuje → "correct"
5. app.py → onizuka("Uczeń odpowiedział poprawnie, pochwal go")
6. Gemini chwali ucznia (2 sek)

7. WAŻNE ZASADY

- Qwen NIGDY nie wymyśla zadań z głowy — ZAWSZE pracuje na materiałach z bazy
- Qwen ZAWSZE dostaje prompty po angielsku — szybciej, krócej, bez dygresji o języku
- Qwen ZAWSZE z think=False — GPU za wolne na myślenie
- Gemini jest nauczycielem — rozmawia, tłumaczy, motywuje, formatuje
- Dane w ChromaDB są rzetelne — pochodzą z oficjalnych arkuszy CKE i podręczników
- Prawa autorskie — materiały lokalne, NIE na GitHubie, NIE publiczne

8. BUDŻET

Realistyczny koszt miesięczny: 5-15 PLN przy codziennej nauce

9. PRZYSZŁOŚĆ (AMADEUS)

Ta sama architektura skaluje się do Amadeusa:
- Gemini = front/myśliciel (bez zmian)
- Qwen = orkiestrator/router (klasyfikacja → satelity)
- Satelity: Claude API (kod), Grok API (dane), ChatGPT API (konsultacje)
- Upgrade GPU (24+ GB VRAM) → Qwen 14B z think=True → Qwen wraca jako mózg
- Modularność kodu pozwala zamienić role bez przepisywania
